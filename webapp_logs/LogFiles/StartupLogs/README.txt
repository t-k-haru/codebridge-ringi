Azure App Service — Container Startup Logs
===========================================

These logs are automatically generated by Azure App Service to help you
debug container startup issues.

Each file contains a merged, chronologically-sorted view of:
  - Platform lifecycle events (image pull, container create, health check)
  - Your container's stdout/stderr output during startup

File naming: {date}_{machine}_{success|failure}.log

  date     — The date the startup occurred (yyyy_MM_dd)
  machine  — The VM instance that ran the container
  success  — The container started and passed health checks
  failure  — The container failed to start or did not pass health checks

To disable these logs, add this app setting:
  WEBSITE_DISABLE_CONTAINER_STARTUP_LOGS = true
